# Projects Subsystem

Project CRUD with sections, five tabbed views (Overview, List, Board, Calendar, Automations), and cascade delete orchestration. Projects are the top-level organizational unit â€” every task, section, and automation rule belongs to a project.

## Overview

Projects group tasks into sections and expose multiple view modes via a tabbed interface. `ProjectService` handles creation (with 3 default sections) and cascade deletion (rules â†’ tasks â†’ sections â†’ project). `SectionService` handles section lifecycle including reassignment of tasks on delete and broken-rule detection. Both services are singletons wired in `serviceContainer.ts` and exposed through `dataStore` actions.

## Services

### ProjectService

Constructor-injected dependencies: `ProjectRepository`, `SectionRepository`, `TaskService`, `TaskRepository`, `AutomationRuleRepository`.

| Method | Purpose |
|--------|---------|
| `createWithDefaults(project)` | Creates project + 3 default sections ("To Do", "Doing", "Done") |
| `static create({ name, description, viewMode })` | Factory: generates UUID + timestamps, returns `Project` object |
| `cascadeDelete(projectId)` | Full cascade: rules â†’ top-level tasks (via TaskService) â†’ sections â†’ project |

**Cascade Delete Order:**
```
cascadeDelete(projectId)
  â†’ automationRuleRepo.findByProjectId() â†’ delete each rule
  â†’ taskRepo.findByProjectId() â†’ filter top-level only â†’ taskService.cascadeDelete() each
  â†’ sectionRepo.findByProjectId() â†’ delete each section
  â†’ projectRepo.delete(projectId)
```

Critical: Only top-level tasks are cascade-deleted directly â€” `TaskService.cascadeDelete()` handles subtasks and dependencies recursively.

### SectionService

Constructor-injected dependencies: `SectionRepository`, `TaskRepository`, `AutomationRuleRepository`.

| Method | Purpose |
|--------|---------|
| `createWithDefaults(name, projectId, order)` | Factory: generates UUID + timestamps, creates and persists section |
| `cascadeDelete(sectionId)` | Reassigns tasks to "To Do" section â†’ deletes section â†’ detects broken automation rules |

**Section Delete Flow:**
```
cascadeDelete(sectionId)
  â†’ find section â†’ find "To Do" section in same project
  â†’ reassign all tasks with sectionId to "To Do" (or null if not found)
  â†’ sectionRepo.delete(sectionId)
  â†’ detectBrokenRules(sectionId, projectId, automationRuleRepo)
```

Critical: `detectBrokenRules()` (from automations feature) disables rules that reference the deleted section. This is a cross-feature dependency.

### Default Sections

Every new project gets 3 sections with deterministic IDs:

| Section | ID Format | Order |
|---------|-----------|-------|
| To Do | `${projectId}-section-todo` | 0 |
| Doing | `${projectId}-section-doing` | 1 |
| Done | `${projectId}-section-done` | 2 |

Critical: Section IDs are NOT UUIDs for defaults. Zod schema uses `z.string()` (no `.min(1)`) to accommodate this format.

## Components

### ProjectView (orchestrator)

Main project view â€” renders header (inline-editable name, share button, new task button) and `ProjectTabs` with all 5 tab contents. Manages sectionâ†’RuleDialog state for creating automation rules from section context menus.

| Responsibility | Detail |
|----------------|--------|
| Tab routing | URL param `?tab=` synced with `appStore.projectTabs` |
| Task filtering | `useFilteredTasks(projectTasks)` applied to List/Board/Calendar |
| Rule count badge | Subscribes to `automationRuleRepository` for enabled/total counts |
| Sectionâ†’Rule dialog | `handleOpenRuleDialogFromSection()` opens `RuleDialog` with prefill |

### ProjectTabs

Radix `Tabs` wrapper rendering 5 tabs with icons. Automations tab shows:
- Badge with enabled rule count (when > 0)
- Warning icon when total rules â‰Ą 10

| Tab | Icon | Content Source |
|-----|------|----------------|
| Overview | `LayoutDashboard` | `ProjectOverview` |
| List | `List` | `TaskList` (features/tasks) |
| Board | `Columns3` | `TaskBoard` (features/tasks) |
| Calendar | `Calendar` | `TaskCalendar` (features/tasks) |
| Automations | `Zap` | `AutomationTab` (features/automations) |

### ProjectList (sidebar)

Sidebar navigation showing "All Tasks" link + project list with:
- Color dots (8-color cycle: amber, teal, rose, violet, sky, emerald, orange, indigo)
- Task count badges per project
- Progress bars (completed/total percentage)
- Active project highlighted with `border-l-accent-brand`

### ProjectDialog

Create/edit dialog with fields: name (required, validated), description (optional), viewMode (edit-only select). Uses `validateProjectName()` for client-side validation.

### ProjectOverview

Overview tab content: description textarea (auto-save on blur), metadata grid (created, updated, total tasks, completed count/percentage, in-progress count), and danger zone with delete confirmation via `AlertDialog`.

### SectionManager

Section CRUD within a project: inline rename (click-to-edit), delete with confirmation (`window.confirm`), add new section. Uses `sectionService.createWithDefaults()` for creation. Delete delegates to `dataStore.deleteSection()` â†’ `sectionService.cascadeDelete()`.

## Data Flow

### Write Path (Project CRUD)

```
Component â†’ dataStore.addProject(project)
  â†’ projectService.createWithDefaults(project)
    â†’ projectRepo.create(project)
    â†’ sectionRepo.create(section) Ă— 3
  â†’ repo subscriptions â†’ useDataStore.setState({ projects, sections })
```

```
Component â†’ dataStore.deleteProject(id)
  â†’ projectService.cascadeDelete(id)
    â†’ delete rules â†’ delete tasks (via TaskService) â†’ delete sections â†’ delete project
  â†’ repo subscriptions â†’ useDataStore.setState(...)
  â†’ router.push('/') (from ProjectOverview)
```

```
Component â†’ dataStore.updateProject(id, updates)
  â†’ projectRepository.update(id, { ...updates, updatedAt: now })
  â†’ repo subscription â†’ useDataStore.setState({ projects })
```

### Tab State Persistence

```
Tab click â†’ navigateToTab(tab)
  â†’ appStore.setProjectTab(projectId, tab)  // persisted in localStorage
  â†’ router.push(`/?project=${projectId}&tab=${tab}`)
```

Default tab from `appStore.getProjectTab()` is `'overview'` when no stored preference exists.
